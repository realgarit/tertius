# tertius v1 — design & execution spec

> A macOS menu-bar utility that gives the trackpad a true middle-mouse **drag**
> (orbit/pan for Blender / CAD), triggered by **⌥ + two-finger trackpad glide**.
> Solo project, personal use, macOS 26 (Tahoe), Apple Silicon.

This document consolidates the original HANDOFF brief with the resolved open
decisions and the autonomous execution / verification plan. It is the single
source of truth for the build.

## 2. Architecture (Clean Architecture, 4 layers)

Dependency rule: source dependencies point only **inward**. Domain imports
nothing but `Foundation`.

```
tertius/
├── Package.swift                 # 4 library targets + app executable + test targets
├── Sources/
│   ├── Domain/                   # pure: value types, TriggerPolicy, DragStateMachine
│   ├── Application/              # use cases + Ports (protocols)
│   ├── Infrastructure/          # adapters: event tap, CGEvent poster, UserDefaults, AX, SMAppService, GitHub
│   └── App/                      # MenuBarExtra app, Settings UI, composition root, Resources (icons)
├── Tests/
│   ├── DomainTests/              # bulk of tests (pure, headless, runs in CI)
│   ├── ApplicationTests/         # use cases with fake ports
│   └── InfrastructureTests/      # adapter-level where headless-testable
├── scripts/                      # make-signing-cert, package-app, codesign, verify-dr-stability, release
├── .github/workflows/            # ci.yml (build+test on push/PR), release.yml (tag-driven)
└── docs/
```

### Domain (pure)
- `AppSettings` — modifier, inputMode, sensitivity, invertX/Y, enabled, launchAtLogin.
- `Modifier` enum (option/control/command/fn); `InputMode` enum (twoFingerDrag/clickDrag).
- `GestureSample` — normalized: phase, deltaX, deltaY, activeModifiers, timestamp, isMomentum.
- `ScrollPhase` enum (began/changed/ended/momentum/none).
- `PointerCommand` enum (middleDown / middleDrag(dx,dy) / middleUp / none).
- `TriggerPolicy` — pure fn `(GestureSample, AppSettings) -> intent`.
- `DragStateMachine` — pure; consumes phase/modifier transitions → emits `PointerCommand`s.
  Also decides whether the underlying event is **swallowed** (only while dragging).

### Application (use cases + Ports)
- Use cases: `MonitorGesturesUseCase`, `UpdateSettingsUseCase`, `ToggleEnabledUseCase`,
  `CheckForUpdatesUseCase`, `ManageLaunchAtLoginUseCase`.
- Ports (protocols): `InputSource`, `PointerActuator`, `ConfigStore`,
  `AccessibilityAuthorizing`, `UpdateChecking`, `LaunchAtLoginManaging`.

### Infrastructure (adapters)
- `ScrollGestureInputSource: InputSource` — `CGEventTap` on `scrollWheel` + `flagsChanged`;
  re-arms on `tapDisabledByTimeout`/`tapDisabledByUserInput`. **(v1 primary)**
- `ClickDragInputSource: InputSource` — optional `leftMouse*`-gated mode.
- `CGEventPointerActuator: PointerActuator` — posts `otherMouse*` button 2 (middle) down/drag/up.
- `UserDefaultsConfigStore: ConfigStore`.
- `AXAccessibilityAuthorizer: AccessibilityAuthorizing` — wraps `AXIsProcessTrusted*`.
- `GitHubReleasesUpdateChecker: UpdateChecking`.
- `SMAppServiceLaunchManager: LaunchAtLoginManaging`.

### App (composition root)
- SwiftUI `MenuBarExtra`, accessory activation policy (no Dock icon).
- Hand-wired dependency injection. Settings window (§6 of brief). Menu-bar status item.

> **Middle button note:** CoreGraphics middle button = `CGMouseButton.center` →
> `otherMouse*` events carry `mouseEventButtonNumber = 2`. (The brief's prose said
> "button 3"; the correct CG constant is **2**. Verified during implementation.)

## 3. The persistence checkpoint (the #1 risk — automated as far as possible)

The whole signing decision exists to make the Accessibility grant survive
upgrades. The technical invariant is: **the codesign designated requirement (DR)
must be byte-identical across two independent builds signed with the same cert.**
If the DR matches, TCC treats build N+1 as the same app and keeps the grant.

`scripts/verify-dr-stability.sh` (CI + local):
1. Build + sign the `.app` twice (two clean builds, same cert).
2. `codesign --display --requirements -` on both.
3. Assert the DR strings are identical AND of the form
   `identifier "io.github.realgarit.tertius" and ...cert leaf...`.
4. Fail loudly if they differ.

This is fully automatable and runs without a human. **The only part needing a
human** is the final end-to-end confirmation: grant Accessibility once, install a
newer build over it, confirm no re-grant prompt. That is a flagged manual test
(see §5).

## 8. Risks carried forward

- **Fn modifier** flaky / system-grabbed → default ⌥, Fn selectable with inline caveat.
- **Sensitivity** needs calibration (scroll deltas ≠ 1:1 pixels) → slider + sane default.
- **Momentum** events must be ignored so drags end crisply at finger-up.
- **App Sandbox OFF**; **Hardened Runtime** only if notarization ever added (not now).
- **Event taps disabled under load** → handle disable callbacks, re-arm.
- **CI runner Xcode/SDK** must support the macOS 26 SDK (deployment target 26.0).
  Verified empirically at M0/M5 by running CI; runner image selected accordingly.
- **Target-app delta vs absolute** — Blender reads MMB-drag deltas; warp cursor
  alongside synthetic drag if any CAD app needs absolute motion.
